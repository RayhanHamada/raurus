# Package Agent Guide

## Package Context

This package is `@raurus/client`, the type-safe client SDK for the Raurus framework. It targets both browser and server runtimes, wraps `openapi-fetch` with the OpenAPI schema generated from `@raurus/server`, and consumes `@raurus/core` types.

This file extends the root [AGENTS.md](../AGENTS.md).

- The public surface is the `createApiClient({ baseUrl, headers? })` factory in `src/common/index.ts`
- The `paths` type from `openapi.gen.ts` is passed as the generic to `openapi-fetch`, so consumers get fully typed request/response shapes
- `openapi.gen.ts` is regenerated by `bun run typegen`, which boots a `@raurus/server` runtime and feeds its OpenAPI JSON through `openapi-typescript`

## Conventions

- `tsconfig.json` includes the `DOM` lib because the SDK is intended to run in the browser as well as on the server
- `tsdown.config.ts` uses a single `src/index.ts` entry; revisit if the package grows adapters under `src/adapters/*/index.ts`
- The `build:before` hook in `tsdown.config.ts` regenerates `openapi.gen.ts` before bundling ‚Äî keep this in place so the published types always match the current server
- Runtime types come from `@raurus/core` ‚Äî do not duplicate domain types here
- `paths` is imported via the `@/*` path alias (resolves to `./src/*`); keep the alias in sync if you add new entry points
- Test files use the `*.test.ts` suffix and live next to the code they cover (see `src/common/index.test.ts`)
- The vitest plugin is enabled via `ultracite/oxlint/vitest` at the root, so use the standard `describe`/`it`/`expect` import shape

## Workflow

- Read the root `AGENTS.md` before planning or implementing changes in this package
- Build with `bun run build` (uses tsdown with exports mode and the `build:before` typegen hook)
- Run tests with `bun run test` (vitest run)
- Type-check with `bun run typecheck`
- Lint and format with `bun run check` / `bun run fix` from the repo root
- Regenerate OpenAPI types with `bun run typegen` whenever `@raurus/server` routes change

## Package Notes

- The current public surface is the `createApiClient` factory wrapping `openapi-fetch` ‚Äî add new runtime helpers under `src/common/` and re-export from `src/index.ts`
- When adding new request shapes, regenerate `openapi.gen.ts` with `bun run typegen` rather than editing it by hand
- Add new runtime deps with `workspace:*` for `@raurus/*` packages; confirm any non-Raurus runtime dep before adding it
